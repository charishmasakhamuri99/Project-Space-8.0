# Systolic Array Image Processing Accelerator

## Project Overview

A 3×3 systolic array hardware accelerator for image convolution (Sobel edge
detection and Gaussian blur). Accepts grayscale pixel values from a `.txt` file
(generated by Python), processes them through the hardware pipeline, and outputs
filtered pixel values to be reconstructed as an image by Python.

## Architecture Summary

```
                   ┌──────────────────────────────────────────────────┐
  pixel_in ──────► │ pixel_input_controller                           │
  (serial)         │  col_cnt, row_cnt, LB write pointer              │
                   └───┬───────┬──────────────────────────┬───────────┘
                       │ LB0   │ LB1                       │ live_pixel
                       ▼       ▼                           │
                   ┌───────┐ ┌───────┐                     │
                   │ Line  │ │ Line  │                     │
                   │ Buf 0 │ │ Buf 1 │                     │
                   │(row   │ │(row   │                     │
                   │ N-2)  │ │ N-1)  │                     │
                   └───┬───┘ └───┬───┘                     │
                       │         │                          │
                       └────┬────┘                          │
                            ▼                               ▼
                   ┌─────────────────────────────────────────────────┐
                   │              sliding_window (3×3)                │
                   │  3 shift-register chains, 9 FF outputs           │
                   └──────────────────────┬──────────────────────────┘
                                          │ w00..w22 (9 pixels)
                                          ▼
                   ┌─────────────────────────────────────────────────┐
                   │           systolic_array_3x3 (9 PEs)            │
                   │  All 9 PEs multiply pixel×weight simultaneously  │
                   └──────────────────────┬──────────────────────────┘
                          ▲               │ p0..p8 (9 products)
                   weight_rom             ▼
                   (Gaussian/Sobel) ┌──────────────────────────────────┐
                                    │     adder_tree (4-level pipe)    │
                                    │     Latency = 4 cycles           │
                                    └──────────────┬───────────────────┘
                                                   │ 27-bit sum
                                                   ▼
                                    ┌──────────────────────────────────┐
                                    │       output_normalizer          │
                                    │  Sobel: |x|→clamp  Gaussian: >>4│
                                    └──────────────┬───────────────────┘
                                                   │ 8-bit pixel
                                                   ▼
                                    ┌──────────────────────────────────┐
                                    │       output_buffer              │
                                    │  256-entry row accumulator       │
                                    └──────────────┬───────────────────┘
                                                   │ pixel_out (serial)
```

**control_unit FSM** drives all enable/address signals to every module.

## Key Design Decisions

| Decision | Choice | Reason |
|---|---|---|
| Convolution mode | Weight-stationary broadcast | Kernel is fixed per frame; all 9 PEs fire in parallel |
| Array size | 3×3 | Matches kernel exactly; 1 output pixel per clock |
| Reduction | Adder tree (4-level) | log₂(9) latency vs 8-cycle chain |
| Line buffer type | Sync SRAM (not shift reg) | Requires random-address read by column index |
| Border handling | Zero-pad in Python | Hardware has no border logic; simpler design |
| Input image | 256×256 → padded to 258×258 | Output = 256×256 |
| Pixel width | 8-bit unsigned | Standard grayscale |
| Weight width | 8-bit signed | Sobel needs negative values |
| Accumulator | 24-bit → 27-bit at tree output | Prevents overflow across 9 additions |

## Complete RTL-to-GDS Flow

### Step 1 — Python Preprocessing

```bash
python python/preprocess.py my_photo.jpg --out pixels_in.txt
# Produces: pixels_in.txt  (66,564 values, 258×258 zero-padded)
```

### Step 2 — RTL Simulation (iverilog — free)

```bash
# Install iverilog: sudo apt install iverilog   (Ubuntu/Debian)
#                   brew install icarus-verilog  (macOS)

make sim          # compile + simulate (uses 10×10 test image internally)
make wave         # open waveforms in GTKWave
make lint         # Verilator lint check
```

Expected simulation output:
```
=== SOBEL-X TEST ===
Image: 10×10 (including zero padding)
Expected output: 8×8
PASS: Output pixel count correct.
...
Simulation complete.
```

### Step 3 — Gate-Level Synthesis (Synopsys DC)

1. Edit `syn/synthesize.tcl` — set `PDK_LIB_PATH` and `TARGET_LIB` for your PDK.
2. Run:
   ```bash
   make synth
   # OR directly:
   dc_shell -f syn/synthesize.tcl | tee syn/dc_run.log
   ```
3. Check outputs in `syn/outputs/`:
   - `top_netlist.v`  — gate-level netlist
   - `top.sdf`        — timing back-annotation
   - `top_pnr.sdc`    — constraints for P&R
   - `timing_setup.rpt`, `area.rpt`, `power.rpt`

**Target frequency:** 100 MHz (10 ns). Achievable on 180nm or better.

**Estimated area (180nm):** ~15,000–25,000 gate equivalents.

### Step 4 — Place and Route (Cadence Innovus / Synopsys ICC2)

```tcl
# In Innovus:
read_netlist syn/outputs/top_netlist.v
read_sdc     syn/outputs/top_pnr.sdc

# Floorplan
create_floorplan -core_aspect_ratio 1.0 -core_utilization 0.7

# Power planning
create_power_plan ...

# Placement
place_design

# CTS (Clock Tree Synthesis)
ccopt_design

# Routing
route_design

# Sign-off
verify_drc
verify_lvs

# Write GDS
write_gds top_final.gds
```

### Step 5 — Sign-off Checks

| Check | Tool | File |
|---|---|---|
| DRC | Cadence Pegasus / Mentor Calibre | top_final.gds |
| LVS | Cadence Pegasus / Mentor Calibre | top_final.gds vs top_netlist.v |
| STA | Synopsys PrimeTime | top_netlist.v + top.sdf |
| Power | Synopsys PrimePower | VCD from gate-level sim |

### Step 6 — Python Postprocessing

```bash
# After hardware run collects output to pixels_out.txt:
python python/postprocess.py pixels_out.txt --out result.png --compare my_photo.jpg --mode sobel
# Saves result.png and prints MAE vs scipy reference
```

## Timing Budget (100 MHz target)

```
Critical path analysis:
  pixel_in → [pe.v multiply] → [adder_tree level 1] = ~4–5 ns (well within 10ns)
  Longest path likely in control_unit FSM decode logic (~3–4 ns)
  Adder tree is pipelined → no long combinational path

Latency from first valid window to first output pixel:
  1 cycle (PE multiply) + 4 cycles (adder tree) + 0 cycles (comb normalizer)
  = 5 clock cycles = 50 ns at 100 MHz
```

## Common Issues and Fixes

| Issue | Cause | Fix |
|---|---|---|
| Output pixel count wrong | control_unit column counter off-by-one | Check `OUTPUT_SIZE-1` boundary |
| All outputs zero | load_weight not reaching PEs | Check `cu_load_weight` routing |
| Wrong values after row 0 | LB ping-pong pointer bug | Trace `lb_wr_sel` in waveform |
| Adder tree sum wrong | Sign extension missing | Check `{{1{p0[23]}}, p0}` syntax |
| Timing violation in DC | Weight ROM combinational path too long | Register ROM outputs in PE weight_reg |
